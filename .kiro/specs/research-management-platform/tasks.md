- [ ] 1. プロジェクト基盤セットアップ

- [x] 1.1 データベーススキーマとSupabase接続を構成する
  - `institutions`、`themes`、`investors` テーブルを論理削除カラム付きで作成する

  - Supabase クライアントをサーバーサイド専用ファイルとして初期化する（クライアントコンポーネントへの漏洩を防ぐ）
  - _Requirements: 1.1, 2.1, 3.1, 4.5, 5.2_

- [x] 1.2 Zodバリデーションスキーマと共通型定義を整備する
  - `Institution`、`Theme`、`Investor`、`Comment` のドメイン型を定義する
  - 各エンティティのZodスキーマ（作成・更新用）を実装する
  - `ActionResult<T>` 型（成功/エラーの判別共用体）を定義する
  - `CommentCategory`、`ThemeStatus`、`InvestorType` の文字列リテラル型を定義する
  - _Requirements: 1.5, 2.5, 3.5, 5.4_

- [ ] 2. レイアウト・共通UIコンポーネントの実装

- [x] 2.1 グローバルレイアウトとサイドバーナビゲーションを構築する
  - サイドバーに研究機関・研究テーマ・投資家・コメント集計の4リンクを配置する
  - 現在のルートに `aria-current="page"` を付与しアクティブ状態を視覚化する（DADSプライマリブルー）

  - `<main id="main-content">` でスキップリンクのターゲットを設定する

- [x] 2.2 フィードバックUIコンポーネント群を実装する
  - `Toast` コンポーネントを `aria-live="polite"` 付きで実装し、3〜5秒で自動消去する
  - `Skeleton` コンポーネント（カード・テーブル行・フォーム向け）を実装する
  - `DeleteConfirmDialog` モーダルを `role="dialog"` + `aria-modal="true"` + フォーカストラップで実装する

- [ ] 3. 研究機関管理の実装

- [x] 3.1 研究機関一覧ページを実装する
  - 機関名・所在地・登録日を表示するテーブルをServer Componentとして実装する
  - 名称・登録日でのソートをURLクエリパラメーターで管理する（`?sort=name&order=asc`）
  - 論理削除済み（`deleted_at IS NOT NULL`）を除外した一覧を取得する
  - 「新規登録」ボタンを配置し登録フォームへ遷移する
  - データ取得中は `Skeleton` を表示し、0件時は空状態メッセージとCTAを表示する
  - _Requirements: 1.4, 7.1, 7.5_

- [x] 3.2 研究機関詳細ページを実装する
  - 機関情報（名称・所在地・説明・連絡先）と更新日時を表示する
  - 関連付けられた研究テーマ・投資家の一覧を同一ページに表示する（6.1の内容）
  - 編集・削除ボタンを配置する（削除は `DeleteConfirmDialog` を経由する）
  - _Requirements: 1.6, 1.3_

- [x] 3.3 研究機関の登録・編集フォームとServer Actionを実装する
  - `useActionState` を使い、Server Actionの結果をフィールド直下のエラーとして表示する
  - 機関名（必須・最大200文字）、所在地、説明（最大2000文字）、連絡先を入力可能にする
  - 送信中はボタンをdisabledにしローディング状態を表示する
  - 成功後は `revalidatePath` でキャッシュを無効化し一覧ページへリダイレクトする
  - `Toast` で成功通知を表示する
  - _Requirements: 1.1, 1.2, 1.5, 7.3_

- [ ] 7. コメント・レビュー機能を実装する

- [x] 7.1 コメント投稿フォームと管理Server Actionを実装する
  - コメント本文（必須・最大1000文字）、種別セレクト（改善要望/不具合/その他）、評価スコア（1〜5のスター評価、任意）を含むフォームを実装する
  - `page_id`（対象画面の識別子、hidden input）と `page_label` を送信してコメントと紐付ける
  - `createComment` Server Actionを `commentSchema` Zodバリデーション付きで実装する
  - `updateComment` / `deleteComment` Server Actionを実装し、コメント作成者が自身のコメントを編集・削除できるUIを提供する
  - 送信中はフォームをdisabledにし、成功後はフォームをリセットして `Toast` を表示する
  - _Requirements: 5.2, 5.3, 5.4, 5.6, 7.3_

- [x] 7.2 コメント一覧表示と各管理画面への統合を行う
  - `CommentSection` コンポーネント（`CommentList` + `CommentForm`）を実装する
  - コメントを投稿日時の降順で表示し、種別バッジとスコア（★表示）を視覚化する
  - バリデーションエラーは `aria-live="polite"` でスクリーンリーダーに通知する
  - 研究機関・研究テーマ・投資家の各一覧ページおよび詳細ページに `CommentSection` を組み込む
  - 各ページの `page_id` を `"{entity_type}/{list|detail}/{id}"` 形式で設定する
  - _Requirements: 5.1, 5.5, 5.4_

- [ ] 8. コメント集計ダッシュボードを実装する

- [x] 8.1 ダッシュボードページのフィルタリングとコメント一覧を実装する
  - 対象画面・種別・投稿日付範囲でフィルタリングできるフォームを実装する（URLクエリパラメーターで状態管理）
  - フィルター条件に合致する全コメントをページネーション付き一覧テーブルで表示する
  - コメント行クリックで `CommentDetailModal` を開き、詳細（本文・種別・対象画面・投稿日時）を表示する
  - _Requirements: 6.1, 6.2, 6.5_

- [x] 8.2 統計サマリーとグラフ可視化を実装する
  - DBの GROUP BY クエリで画面別コメント件数・平均スコアを集計し `StatsSummary` コンポーネントで表示する

  - _Requirements: 6.3, 6.4_

- [x] 8.3 CSVエクスポート機能を実装する

  - エクスポートするCSVには対象画面・種別・スコア・本文・投稿日時を含める
  - _Requirements: 6.6_
